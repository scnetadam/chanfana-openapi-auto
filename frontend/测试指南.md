# 支付宝小程序测试指南

## 一、环境配置

### 1. 后端服务
```bash
# 确认后端服务运行中
访问：http://192.168.0.100:3000/health
应返回：{"service":"龟钮印信 API","version":"3.3.0","status":"ok"...}
```

### 2. 小程序配置
- API地址：`http://192.168.0.100:3000`
- 超时时间：30秒
- 登录方式：昵称登录（简化版）

## 二、支付宝小程序开发者工具配置

### 步骤1：导入项目
1. 打开支付宝小程序开发者工具
2. 点击"打开项目"或"导入项目"
3. 选择目录：`D:\X402-DEVECO\frontend\dist\build\mp-alipay`
4. 项目名称：龟钮印信

### 步骤2：配置开发者工具（重要！）
1. 点击右上角"详情"按钮
2. 在"本地设置"标签页中，勾选以下选项：
   - ✅ 不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书
   - ✅ 启用自定义处理网络请求
   - ✅ 不校验安全域名、TLS版本以及HTTPS证书

### 步骤3：重新编译
1. 点击工具栏的"编译"按钮
2. 或按快捷键 Ctrl+B
3. 等待编译完成

## 三、测试登录

### 登录步骤
1. 在模拟器中看到登录页面
2. 输入昵称（如：测试用户）
3. 点击"登录"按钮
4. 等待响应（最多30秒）

### 查看日志
1. 点击底部的"调试器"标签
2. 选择"Console"控制台
3. 查看以下日志：
   - `[API] request: POST http://192.168.0.100:3000/api/auth/login`
   - `[API] response: 200 ...`

### 如果仍然超时

#### 方案1：检查网络连接
```bash
# 在命令行测试API
curl http://192.168.0.100:3000/health
# 或使用PowerShell
Invoke-WebRequest -Uri "http://192.168.0.100:3000/health"
```

#### 方案2：使用localhost
如果192.168.0.100不通，尝试：
1. 修改 `D:\X402-DEVECO\frontend\.env`
2. 改为：`VITE_API_BASE=http://localhost:3000`
3. 重新构建：`npm run build:mp-alipay`
4. 在开发者工具中重新导入项目

#### 方案3：使用127.0.0.1
如果localhost不通，尝试：
1. 修改 `.env` 为：`VITE_API_BASE=http://127.0.0.1:3000`
2. 重新构建

#### 方案4：真机调试
1. 点击工具栏的"真机调试"按钮
2. 使用支付宝APP扫码
3. 在真机上测试（真机网络环境更稳定）

## 四、常见问题

### Q1: 提示"request:fail"
**原因**：域名未配置或不支持HTTP
**解决**：确保勾选"不校验合法域名"选项

### Q2: 提示"超时"
**原因**：网络不通或后端未启动
**解决**：
1. 确认后端服务运行：`netstat -ano | findstr :3000`
2. 测试API连接：访问 http://192.168.0.100:3000/health

### Q3: 登录成功但无法跳转
**原因**：TabBar配置问题
**解决**：检查控制台错误日志

### Q4: 控制台无任何日志
**原因**：代码未执行或被缓存
**解决**：
1. 点击"清缓存" -> "全部清除"
2. 重新编译

### ⚠️ 待解决问题

**API兼容性问题**: `my is not defined`
- 原因：vendor.js包含54处支付宝API调用（my.xxx）
- 位置：common/vendor.js
- 影响：运行时错误

## 问题根源

uni-app编译时默认使用支付宝平台API：
```javascript
// vendor.js中的代码
my.getSystemInfoSync()      // 支付宝API
my.setStorage()             // 支付宝API
my.SDKVersion               // 支付宝API
my.EventChannel             // 支付宝API
my.$onErrorHandlers         // 支付宝API
```

微信小程序需要：
```javascript
wx.getSystemInfoSync()      // 微信API
wx.setStorage()             // 微信API
wx.SDKVersion               // 微信API
// 等等...
```

### 方案二：手动替换API（不推荐）

手动修改vendor.js中的API调用：

```javascript

my.getSystemInfoSync() → wx.getSystemInfoSync()
my.setStorage() → wx.setStorage()
my.SDKVersion → wx.SDKVersion
// ... 54处替换
```

#### 缺点

- ❌ 工作量大（54处替换）
- ❌ 容易出错
- ❌ 不易维护
- ❌ 后续更新困难

### 方案三：使用条件编译

修改源码，添加条件编译：

```javascript
// #ifdef MP-WEIXIN
const api = wx
// #endif

// #ifdef MP-ALIPAY
const api = my
// #endif

api.getSystemInfoSync()
```

#### 缺点

- ❌ 需要修改源码
- ❌ 工作量大
- ❌ 需要重新编译

## 常见问题

### Q1: 为什么不直接修改vendor.js？

A: vendor.js是编译产物，每次构建都会重新生成，修改会被覆盖。

### Q2: HBuilderX和命令行构建有什么区别？

A: HBuilderX内置了平台适配逻辑，会自动处理API差异和配置差异。

### Q3: 能否同时支持支付宝和微信？

A: 可以！uni-app一套代码多端发布，使用条件编译处理平台差异。

### Q4: 构建后还需要修改吗？

A: 使用HBuilderX构建后，通常不需要手动修改，开箱即用。

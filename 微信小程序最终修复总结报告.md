
## 问题根源

### 为什么会有API兼容性问题？

当前微信小程序构建产物是通过以下方式生成的：
1. 使用uni-app CLI构建支付宝小程序
2. 手动转换文件格式和配置
3. **但vendor.js仍包含支付宝API调用**

### vendor.js中的问题代码

```javascript
// 支付宝API调用
my.getSystemInfoSync()      // 获取系统信息
my.setStorage()             // 本地存储
my.SDKVersion               // SDK版本
my.EventChannel             // 事件通道
my.$onErrorHandlers         // 错误处理
```

微信小程序需要：
```javascript
// 微信API调用
wx.getSystemInfoSync()
wx.setStorage()
wx.SDKVersion
// 等等...
```

### 方案二：手动修复vendor.js（不推荐）

需要手动替换54处API调用：

```javascript
// 需要替换的模式
my.getSystemInfoSync → wx.getSystemInfoSync
my.setStorage → wx.setStorage
my.SDKVersion → wx.SDKVersion
// ... 54处
```

#### 缺点

- ❌ 工作量大（54处替换）
- ❌ 容易出错
- ❌ 不易维护
- ❌ 后续构建会被覆盖
- ❌ 可能遗漏某些API

### 方案三：修改源码添加条件编译（不推荐）

需要修改所有使用平台API的源码：

```javascript
// #ifdef MP-WEIXIN
const result = wx.getSystemInfoSync()
// #endif

// #ifdef MP-ALIPAY
const result = my.getSystemInfoSync()
// #endif
```

#### 缺点

- ❌ 需要修改大量源码
- ❌ 工作量巨大
- ❌ 维护成本高
- ❌ 容易遗漏

# 解决API超时问题 - 完整步骤

## 当前状态
- ✅ 后端服务运行中：端口3000
- ✅ API测试成功：localhost可访问
- ✅ 小程序已构建：使用localhost配置
- ❌ 支付宝小程序开发者工具：未运行

## 问题原因
支付宝小程序开发者工具可能无法直接访问localhost，因为：
1. 开发者工具运行在独立进程/沙箱环境
2. 网络请求可能被限制
3. 需要特殊配置才能访问本地服务

## 解决方案

### 方案1：使用浏览器测试API（推荐先做）

1. **打开测试页面**
   - 用浏览器打开：`D:\X402-DEVECO\test-api.html`
   - 或直接双击该文件

2. **点击测试按钮**
   - 测试1: 健康检查
   - 测试2: 登录接口
   - 测试3: 活动列表
   - 测试4: MAAS大模型列表

3. **查看结果**
   - 如果全部成功 ✅：API服务正常
   - 如果失败 ❌：检查后端服务

### 方案2：正确配置支付宝小程序开发者工具

1. **打开开发者工具**
   - 下载地址：https://opendocs.alipay.com/mini/ide/download
   - 安装并启动

2. **导入项目**
   - 点击"打开项目"
   - 选择：`D:\X402-DEVECO\frontend\dist\build\mp-alipay`

3. **关键配置（必须！）**
   - 点击右上角"详情"
   - 选择"本地设置"标签
   - **勾选以下所有选项**：
     ```
     ✅ 不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书
     ✅ 启用自定义处理网络请求
     ✅ 不校验安全域名、TLS版本以及HTTPS证书
     ```

4. **清除缓存**
   - 点击工具栏"工具" -> "清缓存" -> "全部清除"
   - 重新编译：Ctrl+B

5. **测试登录**
   - 输入昵称
   - 点击登录
   - 查看控制台日志

### 方案3：使用真机调试（最可靠）

1. **在开发者工具中**
   - 点击工具栏"真机调试"按钮
   - 生成二维码

2. **用支付宝APP扫码**
   - 打开支付宝APP
   - 扫描二维码
   - 在真机上测试

3. **真机优势**
   - 网络环境真实
   - 不受开发者工具限制
   - 更接近生产环境

### 方案4：修改为实际IP地址

如果localhost仍然不通，使用实际IP：

1. **查看本机IP**
   ```powershell
   ipconfig | findstr "IPv4"
   ```
   结果：192.168.0.100

2. **修改配置**
   ```bash
   # 编辑 D:\X402-DEVECO\frontend\.env
   VITE_API_BASE=http://192.168.0.100:3000
   ```

3. **重新构建**
   ```bash
   cd D:\X402-DEVECO\frontend
   npm run build:mp-alipay
   ```

4. **重新导入项目**
   - 在开发者工具中重新导入
   - 或点击"编译"重新加载

### 方案5：检查防火墙

1. **临时关闭防火墙测试**
   - Windows设置 -> 更新和安全 -> Windows安全 -> 防火墙和网络保护
   - 关闭防火墙（仅用于测试）

2. **或添加端口例外**
   - 允许端口3000入站连接

## 常见错误

### 错误1: "request:fail url not in domain list"
**原因**：未勾选"不校验合法域名"
**解决**：在"详情" -> "本地设置"中勾选

### 错误2: "request:fail timeout"
**原因**：网络不通或后端未启动
**解决**：
1. 测试API连接（用浏览器打开test-api.html）
2. 确认后端运行（netstat -ano | findstr :3000）

### 错误3: "request:fail"
**原因**：无法连接到服务器
**解决**：
1. 检查API地址配置
2. 尝试使用实际IP地址
3. 使用真机调试

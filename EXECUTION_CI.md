# 龟钮印信 · 执行系统CI规范

> 版本: v3.3.0 | 更新: 2026-07-17
>
> 本文档定义龟钮印信执行系统的全套文案CI规范——从用户触达、交互引导、状态反馈到运营推送的完整语言体系。
> 所有前端页面(HarmonyOS .ets + 小程序 .vue)的展示文案必须遵照本规范，禁止随意编写。

---

## 一、文案总则

### 1.1 品牌语感

| 维度 | 规范 | 示例 |
|------|------|------|
| **人称** | 第二人称「你」为主，第一人称「我」仅限「我的」页面 | 「你已获得推广金 ¥12.80」 |
| **语气** | 自信、干练、有仪式感。不卑不亢，不说教 | ✅「存证已上链」 ❌「恭喜您成功上链了哦」 |
| **节奏** | 信息先行，动词开头。短句≤12字，长句拆分 | ✅「任务完成 · 佣金结算中」 ❌「您的任务已经完成并且佣金正在结算中」 |
| **品牌印记** | 关键场景植入品牌符号(龟/印/存证) | 「[龟] 存证追踪」「印信结算」 |
| **数字** | 金额带¥，百分比带%，数值用千分位 | ¥1,200.00 / 12.8% / 1,280次 |

### 1.2 禁用词表

| 禁用 | 替换 | 原因 |
|------|------|------|
| 请/请您 | 直接动词 | 冗余 |
| 成功了/了呢 | 删去助词 | 幼稚 |
| 稍等/等一下 | 加载中…/处理中… | 不专业 |
| 出错了 | 具体错误描述 | 含糊 |
| 暂无数据 | 空状态品牌文案(见§六) | 冷漠 |
| 敬请期待 | 即将上线 · [龟]预留 | 空洞 |
| 联系客服 | AI助手帮你解决 | 与品牌AI定位冲突 |

### 1.3 标点规范

| 规则 | 示例 |
|------|------|
| 列表项用 · (中圆点)分隔 | 「推广 · 试驾 · 结算」 |
| 状态对应用 · 连接 | 「任务完成 · 佣金到账」 |
| 时间戳用 · 连接 | 「2026-07-17 · 14:30」 |
| 金额/数字不顿号 | 「¥50.00 / ¥120.00」 |
| 品牌符号后空格 | 「[龟] 存证追踪」 |
| 英文标识前空格 | 「KOL · 实时加权」 |

## 十五、跨端一致性校验

### 15.1 HarmonyOS vs 小程序文案对照

所有文案必须两端完全一致。校验规则:

| 校验项 | 说明 | CI检查 |
|--------|------|--------|
| 按钮文案 | ets和vue中同一按钮文案一致 | grep比对 |
| 空状态文案 | 两端空状态主文案+副文案一致 | 人工 |
| 错误提示 | 同一错误码对应相同文案 | grep比对 |
| 通知模板 | 推送文案两端统一 | 人工 |
| 数字格式 | 千分位/小数位/货币符号一致 | grep比对 |

### 15.2 文案CI自动化检查项

```yaml
# ci.yml quality-gate 新增步骤

- name: Copy consistency check
  run: |
    # 按钮文案一致性
    grep -rn "发布内容\|确认结算\|提交预约\|接受任务" entry/src/main/ets/pages/ --include="*.ets" | sort > /tmp/ets-btn.txt
    grep -rn "发布内容\|确认结算\|提交预约\|接受任务" frontend/src/pages/ --include="*.vue" | sort > /tmp/vue-btn.txt
    echo "=== ETS buttons ===" && cat /tmp/ets-btn.txt
    echo "=== Vue buttons ===" && cat /tmp/vue-btn.txt
    
    # 空状态文案一致性
    grep -rn "暂无" entry/src/main/ets/pages/ --include="*.ets" | sort > /tmp/ets-empty.txt
    grep -rn "暂无" frontend/src/pages/ --include="*.vue" | sort > /tmp/vue-empty.txt
    
    # 禁用词检查
    grep -rn "请您\|稍等\|出错了\|暂无数据\|敬请期待" entry/src/main/ets/ frontend/src/ --include="*.ets" --include="*.vue" | head -20 || echo "✓ No forbidden words"
    
    # 品牌符号检查: 关键页面应有[龟]或印信
    for page in LoginPage Index ActivitiesPage SettlementPage HashPage; do
      if ! grep -rq "龟\|印信\|存证" entry/src/main/ets/pages/; then
        echo "::warning::Missing brand symbol in $page"
      fi
    done
```

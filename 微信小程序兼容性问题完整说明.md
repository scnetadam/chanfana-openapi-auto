# 微信小程序兼容性问题完整说明

## 已发现的问题

### 1. page.json配置问题 ✅ 已修复

**问题**: `无效的 page.json ["defaultTitle"]`

**原因**: 支付宝小程序使用 `defaultTitle`，微信小程序使用 `navigationBarTitleText`

**修复**: 已批量替换所有30个page.json文件

```json
// 修复前（支付宝）
{
  "defaultTitle": "页面标题"
}

// 修复后（微信）
{
  "navigationBarTitleText": "页面标题"
}
```

### 2. API兼容性问题 ⚠️ 需注意

**问题**: `my is not defined`

**原因**: 代码中使用了支付宝小程序API（`my.xxx`），微信小程序不支持

**支付宝API → 微信API对照表**:

| 支付宝 | 微信 | 说明 |
|--------|------|------|
| `my.request` | `wx.request` | 网络请求 |
| `my.getStorageSync` | `wx.getStorageSync` | 同步存储 |
| `my.setStorageSync` | `wx.setStorageSync` | 同步存储 |
| `my.showToast` | `wx.showToast` | 提示框 |
| `my.navigateTo` | `wx.navigateTo` | 页面跳转 |

**解决方案**:

uni-app已自动处理API差异，使用 `uni.xxx` 统一API：

```javascript
// 推荐写法（跨平台）
uni.request({ url: '...' })
uni.showToast({ title: '成功' })
uni.navigateTo({ url: '/pages/xxx' })

// 不要直接使用平台API
// my.request()  ❌ 支付宝专用
// wx.request()  ❌ 微信专用
```

### 3. TabBar数量限制 ✅ 已修复

**问题**: `tabBar.list 不能超过 5 项`

**修复**: 已将Tab从6个减少到5个

### 4. 文件格式问题 ✅ 已修复

**问题**: 缺少.wxml文件

**修复**: 已将.axml转换为.wxml，.acss转换为.wxss
